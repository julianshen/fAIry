# Bundle the Pi agent (M6-1) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** The `.app` ships a `bun --compile`d Pi binary (`fairy-pi`) and the daemon spawns it via a `FAIRY_PI_BIN` override, so a packaged app needs no external `pi` on `PATH`.

**Architecture:** Pi (`@earendil-works/pi-coding-agent`, MIT) is pinned as a pi-daemon devDependency and `bun --compile`d to `fairy-pi`. The daemon's `spawn("pi")` becomes `spawn(resolveAssetPath(env, "FAIRY_PI_BIN", "pi"))` — reusing the 5a helper. The shell's `DaemonLocator` sets `FAIRY_PI_BIN` to the bundled binary, and `package.sh` bundles it + its MIT attribution.

**Tech Stack:** Bun (compile + vitest), Swift 6 / SPM (XCTest), Bash. Run `bun` from `packages/pi-daemon/`, `swift`/scripts from `packages/mac-shell/`.

**Spec:** `docs/superpowers/specs/2026-06-10-bundle-pi-design.md` (M6-1; M6-2 = e2e on a real site — the full-agent-turn smoke; M6-3 = CI release).

Derisk (already run): `bun build <pi cli.js> --compile` produced a running 68 MB binary (`--version` printed `0.0.0` — Pi resolves its version from a runtime file the compile doesn't embed; harmless for the daemon, which never reads it). Confirmed sites: `packages/pi-daemon/src/main.ts:114` is `spawn("pi", ["--mode", "rpc", "-e", BROWSER_EXTENSION], …)`; `resolveAssetPath` is imported at `main.ts:15`; `DaemonLocator`'s bundled `environment` already carries `FAIRY_BROWSER_BRIDGE`/`FAIRY_SKILLS_ROOT`/`PATH`.
